### Ответы на контрольные вопросы
**Вопрос 1.** Что такое ViewBinding и в чем его преимущество перед классическим методом findViewById()? (Кратко опишите, как его подключить).

ViewBinding (Привязка представлений) — это функция, позволяющая легко взаимодействовать с UI-компонентами в Android-приложении. При включении, она автоматически создает класс привязки для каждого файла XML-макета, предоставляя прямые ссылки на все представления с ID внутри этого макета.

Чтобы включить View Binding в модуле надо добавить элемент в файл build.gradle:
    android {
        ...
        viewBinding {
            enabled = true
        }
    }


**Вопрос 2.** В чем разница между декларативной (XML-атрибут onClick) и программной (setOnClickListener) подпиской на событие? Когда какой способ предпочтительнее?

Декларативная подписка связывает нажатие с методом Activity, упрощая прототипирование, но нарушая инкапсуляцию. Программная более гибкая, безопасная и предпочтительна для поддержки, динамических интерфейсов и фрагментов. 

**Вопрос 3.** Что произойдет, если в методе-обработчике, указанном в XML, изменить сигнатуру (например, убрать параметр View v)? Почему?

Если в XML-файле макета (layout) через атрибут android:onClick="methodName" указан обработчик нажатия, а в соответствующей Activity (или Fragment) сигнатура этого метода изменена (например, убран единственный параметр View v), произойдет критическая ошибка времени выполнения (runtime crash) — java.lang.IllegalStateException или java.lang.NoSuchMethodException при попытке нажать на элемент.

**Вопрос 4.** Опишите жизненный цикл Activity. В каком методе (onCreate, onStart, onResume) лучше всего инициализировать слушатели для кнопок и почему?

Основные методы жизненного цикла Activity
onCreate(): Вызывается при первом создании активности. Здесь выполняются базовые настройки: инициализация компонентов (findViewById), привязка данных к спискам и установка макета (setContentView). Это происходит один раз за цикл существования.

onStart(): Вызывается сразу после onCreate() или onRestart(). Активность становится видимой для пользователя, но еще не взаимодействует с ним.

onResume(): Активность переходит в активное состояние, становится видимой и получает фокус (взаимодействие с пользователем). Приложение находится здесь, пока пользователь не уйдет с экрана.

onPause(): Активность теряет фокус (например, поверх открылось диалоговое окно или активити частично перекрыта). Здесь стоит приостанавливать тяжелые задачи, анимации или сохранять данные.

onStop(): Активность больше не видна пользователю. Можно освобождать тяжелые ресурсы.

onRestart(): Вызывается, если активность была остановлена (onStop()) и снова запускается.

onDestroy(): Вызывается перед уничтожением активности. Последний шанс освободить ресурсы

Лучше всего инициализировать слушатели для кнопок в методе onCreate(). onCreate() вызывается один раз при создании экземпляра активности. Инициализация слушателей не требует повторения каждый раз, когда пользователь сворачивает и разворачивает приложение.

**Вопрос 5.** Что такое анонимный внутренний класс? Как он используется при установке слушателей событий в Java?

Анонимный внутренний класс в Java — это класс, у которого нет имени, который объявляется и создается (инстанцируется) одновременно в одном операторе. Он используется для однократного переопределения методов существующего класса или реализации интерфейса, не требуя создания отдельного именованного класса. 

Анонимные классы идеально подходят для обработки событий, так как они позволяют смешать код создания компонента с кодом обработки его поведения.
