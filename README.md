### Ответы на контрольные вопросы
1. Понятие виджета.

Виджет (Widget) — это элемент пользовательского интерфейса в Android.
Виджеты являются наследниками класса View (или его потомков) и представляют собой готовые визуальные компоненты, которые можно размещать на экране.
    Примеры виджетов:

TextView — текстовое поле

Button — кнопка

EditText — поле ввода

ImageView — изображение

CheckBox, RadioButton, ProgressBar и т.д.

В лабораторной работе вместо стандартных виджетов мы использовали кастомный виджет — класс DrawView, который сам рисует фигуры на холсте Canvas.

2. Графические примитивы.

Графические примитивы — это базовые геометрические фигуры и элементы, которые можно рисовать на холсте Canvas с помощью класса Paint.
Основные графические примитивы, использованные в работе:

drawRect() — прямоугольник

drawCircle() — круг / овал

drawPath() — произвольная фигура (треугольник, крыша дома и т.д.)

drawLine() / drawLines() — отрезки и линии

drawText() — текст

drawARGB() — заливка фона цветом

Все примитивы рисуются с помощью объекта Paint, в котором задаются цвет, толщина линии, стиль (заливка или контур) и сглаживание.

3. Переопределение метода.

Переопределение метода — это механизм, при котором класс-наследник изменяет реализацию метода, уже существующего в родительском классе.
В лабораторной работе мы переопределили метод:

Kotlinoverride fun onDraw(canvas: Canvas)

Этот метод объявлен в классе View. Мы создали свой класс DrawView, который наследуется от View, и полностью заменили поведение метода onDraw, чтобы вместо стандартного пустого экрана рисовать свои фигуры.
Правило переопределения:
Сигнатура метода (название, параметры, возвращаемый тип) должна полностью совпадать с методом родительского класса.

4. Наследование.

Наследование — один из основных принципов ООП, при котором один класс (дочерний) получает все свойства и методы другого класса (родительского).
В работе использовалось два примера наследования:

class DrawView(context: Context) : View(context)

→ класс DrawView наследует все возможности обычного View и добавляет собственную логику рисования.

class MainActivity : ComponentActivity()

→ MainActivity наследует базовые возможности активности Android.

Благодаря наследованию мы можем:

использовать готовые методы родительского класса (super.onDraw())

переопределять нужные методы

расширять функциональность без копирования кода
